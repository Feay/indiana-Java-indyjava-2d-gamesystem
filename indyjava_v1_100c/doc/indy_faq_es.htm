FAQ:


¿Se necesita algún software especial para echarle un vistazo al juego?

No. Cualquier navegador con soporte Java servirá. Basta cargar la página INDY.HTM y pulsar el botón para jugar.


Tengo problemas ejecutando el juego con Micro$oft Internet Explorer y Netscape Navigator

Me temo que haya problemas de compatibilidad. Por ahora, hay que usar IE 4.0 y Netscape 4.06, o mejor aún, un intérprete Java.
Concretamente, MS Explorer 3.0 y Netscape 4.03 (y versiones anteriores) no lo podrán ejecutar.


¿Por qué no obtengo un soporte completo para música y sonido cuando ejecuto el juego?

Java 1.1 (incluido en todos los navegadores actuales) tiene soporte limitado para sonido. La mejor opción es obtener el runtime de Java 1.2.


¿Por qué no puedo salvar partidas?

Se está ejecutando el juego como un Applet (es decir desde un navegador). El Java de los navegadores pone restricciones de seguridad en los Applets. Hace falta  ejecutar el juego como una Aplicación Java para poder cargar y salvar partidas.


¿Qué necesito para ejecutar el juego como una Aplicación Java?

Necesita conseguir un intérprete Java. Sun ofrece intérpretes Java para casi cualquier sistema. Descargue la última versión desde allíi, o desde uno de los miles de "mirrors". Por ejemplo, el archivo de la versión Windows95/98/NT se llama JRE12-WIN32.EXE y ocupa 9 megas (comprimido).
Entiéndase bien que esto no actualizará la Máquina Virtual Java de su navegador a la versión 1.2. Este intérprete se ejecuta independientemente de cualquier navegador.
Una vez que se tenga correctamente instalado, se podrá ejecutar el juego escribiendo JAVA -jar INDY.JAR en la línea de comandos del Sistema Operativo.
Si se usa w95/98/NT y se instaló Java 1.2, basta hacer doble clic sobre INDY.JAR para lanzar el "engine" como aplicación. No hace falta "pelearse" con la línea de comandos.


¿Necesito estar conectado a la Red para poder jugar?

No. Esto es totalmente off-line. ¡No dispone de multiplayer todavía! Pero podría ser incluido en futuras versiones. ;-)


¿Por qué es tan pequeña la pantalla de juego?

Quería recrear la apariencia de los juegos al "viejo estilo". Por eso la demo está a 320x200. Sin embargo, cuando se crea una aventura propia, se puede especificar cualquier tamaño para la ventana. Mire los tutoriales.


He jugado la demo, y ahora quiero crear mi propia aventura. ¿Desde dónde puedo descargar el engine?

¡Ya lo tiene! Esta es una solución todo-en-uno: la aplicación IndyJava es a la vez el motor del juego y el compilador. Solo se necesita ejecutarlo como aplicación Java, e ir al menu File/Import . ¡Pero tendrá que leee los tutoriales primero!


¿Dónde está la documentación para programar el Engine?

Toda la documentación está en el ZIP. También está en la web.


¿Qué formatos puedo usar para crear aventuras con el motor?

Se pueden usar todos los formatos de archivo soportados por la versión de Java que se esté utilizando.
Para java 1.1 esto incluye *.GIF, *.JPEG, y una variedad concreta de archivos*.AU.
Para java 1.2 esto incluye *.GIF (estáticos y animados), *.JPEG, *.MID, *.WAV, *.AU


¿Qué editores puedo usar para crear estos juegos?

Los scripts se pueden crear con cualquier editor de texto al estilo Notepad. El UltraEdit  también está muy bien.
Para crear y editar gráficos, sugiero Paint Shop Pro.


¿Necesito saber Java para poder crear scripts de aventuras para el engine?

¡De ninguna manera! El engine usa su propio lenguaje de script. No tiene nada en común con el lenguaje en el que está escrito. Echa un vistazo al tutorial.


¡No encuentro 'Import' en la barra de menús!

Se está ejecutando el juego como un Applet (ie. desde un navegador). Para desarrollar scripts se tiene que ejecutar el juego como una Aplicación Java (ver más arriba).
